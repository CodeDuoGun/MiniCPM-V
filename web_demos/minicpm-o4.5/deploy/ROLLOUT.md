# 灰度上线门槛

1. `LOAD_MODEL=false` 完成接口、槽位和前端协议测试。
2. 单 GPU 原版模型 smoke test 通过，连续运行 2 小时无显存增长。
3. 医学专项测试集由至少两位医生复核；急症召回 100%，处方/确诊越界率 0%。
4. 单实例并发固定为 1；更高并发通过一 GPU 一实例横向扩容。网关必须按 `uid` 粘滞。
5. 先内部账号 1%，观察首包延迟、打断延迟、GPU OOM、5xx、槽位完整率与人工接管率。
6. 依次提高到 5%、20%、50%、100%，每级至少观察一个完整业务高峰。
7. 任一红线触发时把 `ROLLOUT_PERCENT` 调为 0，并保留 2.6 服务；不得在活动会话中切模型。

推荐红线：5xx > 0.5%、GPU OOM > 0、急症漏报 > 0、隐私文档转写 > 0、p95 首包延迟超出基线 30%。
